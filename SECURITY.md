# Security Policy

## Supported Versions

This project is actively maintained on the `main` branch.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately to:

- `contact@fruity-ai.com`

When possible, include:
- A clear description of the issue
- Reproduction steps
- Potential impact
- Suggested remediation (optional)

Please do not open public GitHub issues for sensitive security findings.

## Response Expectations

Fruity-AI will acknowledge receipt as quickly as possible and prioritize triage based on impact and exploitability.

## Scope Notes

Security reports should focus on:
- Credential exposure or unsafe defaults
- Unauthorized access paths
- Data leakage risks
- Vulnerabilities in agent execution, skill boundaries, or transport handling
